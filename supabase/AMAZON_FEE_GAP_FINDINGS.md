# Amazon Fee Gap Analysis & Backfill

**Date:** 2026-03-05
**Analyst:** Claude Code (automated)

## Summary

1,468 Amazon orders had no `channel_fees` records. After backfilling settlement data, **540 orders were resolved** (1,598 new fees created + 168 retried = 1,766 fee rows added).

## Root Cause

`amazon-settlements.mjs` fetched only the **last 20 reports** per run (`pageSize=20`) and used a cursor that tracked the most recently processed report. Settlement reports for Oct–Dec 2025 were generated by Amazon in Dec 2025/Jan 2026 (batch-generated), causing the cursor to skip over them.

**Fix applied:** Updated `amazon-settlements.mjs` to:
- Paginate all available reports when `--all` is used (up to 100 per page)
- Re-check the last 5 reports on each run even when cursor matches (catches out-of-order generation)
- Fix nextToken pagination bug (was sending extra params with nextToken)

## The Oct–Nov 2025 Gap

A 29-day gap exists where **no settlement report was generated** by Amazon:
- Last settlement before gap: ends **2025-10-27** (settlement 24844117711)
- First settlement after gap: starts **2025-11-25** (settlement 25019567921, only 2 rows)

This affects ~198 orders (136 in Oct, 62 in Nov). Possible causes:
1. Amazon account hold or disbursement delay
2. Settlement consolidation into a later period
3. Reports purged from SP-API (unlikely — other old reports still exist)

**Recommended action:** Check Amazon Seller Central > Reports > Payments > Settlement Reports for this period manually. If a report exists there but not via SP-API, download and import manually.
